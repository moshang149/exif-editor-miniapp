### 1.2 设计原则

1. **客户端处理优先**: EXIF 解析与合成在客户端完成，避免上传原图——既保护隐私，又加速体验
2. **层级解耦**: 视图层不直接调用微信 API，统一经过 Utils → Service 层
3. **组件化**: 复用组件独立封装，通过 properties/events 与父页面通信
4. **模板驱动**: 模板数据是核心数据模型，编辑器、预览、保存都围绕模板展开

## 二、核心技术选型与理由

### 2.1 EXIF 处理方案

| 方案 | 描述 | 优劣 |
|------|------|------|
| **客户端 Canvas 方案** ✅ | 读取 EXIF → Canvas 重绘图片 → 将新 EXIF 写入 → 导出 | 隐私安全、离线可用、无服务端成本。**局限**: 微信小程序 Canvas 不支持直接写入完整 EXIF 二进制数据 |
| **服务端方案** | 上传图片 → 服务端处理 EXIF → 下载处理后的图片 | 功能完整、支持所有 EXIF 标签。**局限**: 需上传原图、隐私风险、有服务端成本 |

**最终选型: 客户端为主 + 服务端为辅**

- 常规场景（修改少量字段）: 客户端 Canvas 处理，在导出的 JPEG 中嵌入 EXIF 数据
- 复杂场景（需要精确二进制 EXIF 操作）: 预留服务端接口，作为降级方案

### 2.2 EXIF 数据流

```
原始图片
   ↓
wx.chooseMedia / wx.chooseImage     ← 获取图片临时路径
   ↓
wx.getImageInfo                      ← 读取基本图片信息（部分 EXIF）
   ↓
Canvas.drawImage                     ← 将图片绘制到离屏 Canvas
   ↓
Canvas.getImageData                  ← 获取像素数据
   ↓
exif.js (客户端 EXIF 库)             ← 解析/构建 EXIF 二进制数据
   ↓
Canvas.toTempFilePath                ← 导出为新图片（含新 EXIF? 微信限制）
   ↓
服务端 EXIF 注入（可选）              ← 对导出图片进行 EXIF 二进制注入
   ↓
wx.saveImageToPhotosAlbum            ← 保存到系统相册
```

### 2.3 关键技术点

| 技术点 | 方案 |
|--------|------|
| EXIF 解析 | 客户端纯 JS 实现（基于 exif-js 思路，适配小程序环境） |
| EXIF 写入 | Canvas 导出 + 服务端后处理注入 |
| 模板存储 | `wx.Storage` 本地存储，JSON 格式 |
| 图片压缩 | `wx.compressImage` 在保存前对过大图片进行压缩 |
| 撤销/重做 | 命令模式（Command Pattern），操作栈存储在内存中 |

## 四、数据模型（概要）

### 4.1 核心实体

```
ExifData {
  make: string           // 设备品牌，如 "Canon"
  model: string          // 设备型号，如 "EOS R5"
  software: string       // 处理软件
  lensMake: string       // 镜头品牌
  lensModel: string      // 镜头型号
  focalLength: string    // 焦距，如 "70mm"
  aperture: string       // 光圈，如 "f/2.8"
  shutterSpeed: string   // 快门速度，如 "1/1000"
  iso: number            // ISO 值
  exposureBias: string   // 曝光补偿
  dateTimeOriginal: string // 原始拍摄时间 ISO 8601
  latitude: number       // 纬度
  longitude: number      // 经度
}
```

### 4.2 模板实体

```
Template {
  id: string             // 唯一标识
  name: string           // 模板名称
  category: enum         // "builtin" | "user"
  exifData: ExifData     // 模板存储的 EXIF 值
  icon: string           // 模板图标/缩略图
  createdAt: number      // 创建时间戳
}
```

## 六、非功能性设计

### 6.1 性能策略

| 策略 | 措施 |
|------|------|
| 包体积控制 | 主包压缩后 ≤ 1.5MB；关于/帮助页放入分包 |
| 首屏加载 | 首页仅加载模板缩略图列表（懒加载），不预加载编辑器 |
| 图片处理 | 大图先用 `wx.compressImage` 压缩后再送入 Canvas |
| setData 优化 | 编辑表单数据使用防抖合并更新（300ms） |

### 6.2 隐私与安全

- 图片处理在客户端完成，不上传原图至服务器
- EXIF 数据仅在客户端解析与构建
- 如需服务端处理 EXIF，传输时使用 HTTPS + 临时 Token + 处理后立即删除
- 首次使用相册/相机功能时，明确向用户说明权限用途

### 6.3 兼容性

| 环境 | 最低版本 |
|------|----------|
| 微信基础库 | 2.20.0 |
| iOS | 12.0+ |
| Android | 8.0+ |
| 微信客户端 | 8.0.0+ |

## 八、风险与降级

| 风险 | 概率 | 影响 | 降级方案 |
|------|------|------|----------|
| Canvas 无法写入完整 EXIF | 高 | 保存的图片不含新 EXIF | 启用服务端 EXIF 注入作为后备 |
| 微信审核不通过（隐私） | 中 | 无法上线 | 确保所有权限申请有明确场景说明 |
| 大图 Canvas 处理 OOM | 中 | 编辑器崩溃 | 限制图片尺寸上限，压缩后再处理 |
| API 基础库版本兼容 | 低 | 部分旧版微信不可用 | 设置合理的最低基础库版本 |
