# EXIF照片编辑器 — 项目需求文档

> 版本: v1.0  
> 最后更新: 2026-06-25  
> 状态: 初稿

---

## 一、产品定位

### 1.1 一句话描述
一款**修复图片参数**的微信小程序——用户可以一键修复照片的设备信息、镜头信息、拍摄时间等 EXIF 元数据，通过模板系统实现快捷批量处理。

> **命名约定**: 产品对外宣传统一使用"修复图片参数"，技术文档/代码中可保留 `exif-editor` 项目名。小程序前端所有面向用户的文案（按钮、标题、引导语）均使用"修复"而非"修改/编辑"。

### 1.2 目标用户

| 用户画像 | 核心场景 |
|----------|----------|
| 摄影爱好者 | 统一展示所用器材信息，批量修复样片参数 |
| 社交媒体用户 | 替换/隐藏真实的拍摄信息，保护隐私 |
| 二手器材卖家 | 为样片补充器材信息，辅助交易展示 |

### 1.3 核心价值
- **简单**: 模板一键套用，无需手动逐个填写
- **高效**: 3 步完成修复（选图 → 套模板/微调 → 保存）
- **原生**: 保存效果对标 iPhone 相册体验

## 二、功能需求

### F-001 图片获取

**优先级**: P0（MVP 必须）

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-001-1 | 从手机相册选择照片 | 拉起系统相册选择器，支持单选，支持主流格式（JPG/PNG/HEIC） |
| F-001-2 | 图片加载预览 | 选中后在编辑页展示原图缩略图，显示当前 EXIF 摘要 |

**边界:**
- 图片大小限制 20MB
- 仅支持静态图片，不支持 Live Photo
- 仅支持相册选图，不支持实时拍照
- 需用户明确授权 `scope.writePhotosAlbum`

---

### F-002 EXIF 数据查看

**优先级**: P0

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-002-1 | 查看图片现有 EXIF 信息 | 读取并展示设备型号、镜头型号、拍摄时间、光圈、快门、ISO、焦距、GPS 等 |
| F-002-2 | 无 EXIF 图片的处理 | 识别并提示"此图片无 EXIF 数据，可新建信息"，所有字段默认空 |

---

### F-003 EXIF 数据编辑

**优先级**: P0

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-003-1 | 编辑设备信息 | 支持修改 Make（品牌）、Model（型号）、Software（处理软件） |
| F-003-2 | 编辑镜头信息 | 支持修改 Lens Make、Lens Model、Focal Length（焦距）、Aperture（光圈） |
| F-003-3 | 编辑拍摄参数 | 支持修改 Shutter Speed、ISO、Exposure Bias（曝光补偿） |
| F-003-4 | 编辑拍摄时间 | 支持修改 DateTimeOriginal（原始拍摄时间），使用日期时间选择器 |
| F-003-5 | 编辑 GPS 信息 | 支持修改 Latitude/Longitude（经纬度），通过地图选点或手动输入 |
| F-003-6 | 清除 EXIF | 一键清空所有可编辑的 EXIF 字段 |
| F-003-7 | 撤销/重做 | 支持编辑操作的撤销与重做（最多 20 步） |

**边界:**
- 仅修改用户可见和常用的 EXIF 字段，不暴露所有原始标签
- 受限于微信小程序能力，部分复杂 EXIF 标签需通过服务端处理
- GPS 字段的手动输入需做经纬度格式校验（-180~180, -90~90）

---

### F-004 模板系统

**优先级**: P0

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-004-1 | 预设模板 | 提供 5-8 个内置模板（如"iPhone 15 Pro""Canon EOS R5""Leica M11"），覆盖主流设备 |
| F-004-2 | 一键套用模板 | 点击模板即可将所有字段填充为模板预设值 |
| F-004-3 | 批量套用模板 | 选择模板后进入批量模式：依次选多张照片 → 逐一预览确认 → 一键全部修复保存 |
| F-004-4 | 保存为我的模板 | 用户可将当前编辑的 EXIF 配置保存为个人模板 |
| F-004-5 | 我的模板管理 | 查看、重命名、删除已保存的个人模板 |
| F-004-6 | 模板搜索 | 在模板列表中按设备名/品牌搜索 |

**边界:**
- 内置模板不可删除，个人模板最多 50 个（游客）/ 200 个（登录用户）
- 批量套用时每张照片独立确认，避免误操作
- 模板仅覆盖常用字段（F-003 所列），不存储全部 EXIF 标签

#### F-004-A 登录系统与游客系统设计

模板系统和历史记录（F-006）需要关联用户身份，设计为**双模式**运行：

| 模式 | 触发条件 | 数据存储 | 能力边界 |
|------|----------|----------|----------|
| **游客模式** | 默认，无需登录 | 本地 `wx.Storage` | 模板 ≤ 50 个，历史 ≤ 20 条，不可跨设备同步 |
| **登录模式** | 用户主动点击"登录同步" | 云端存储（通过后端 API） | 模板 ≤ 200 个，历史 ≤ 100 条，多设备同步，解锁社区分享（P2） |

**关键设计原则:**
- **游客可用全部核心功能** — 不强制登录，不阻断核心体验
- **登录为增值，非门槛** — 登录后解锁"跨设备同步"和"更多容量"，而非解锁基础功能
- **静默升级路径** — 登录后自动合并本地数据到云端，不丢失游客期间创建的内容
- **模板社区（P2）前提** — 社区分享功能必须在登录态下进行，游客仅可浏览不可发布

### F-005 图片保存

**优先级**: P0

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-005-1 | 保存至相册 | 修改后的图片保存到系统相册，格式保持原图格式 |
| F-005-2 | 保存进度反馈 | 显示导出进度，大图压缩处理时有 loading 状态 |
| F-005-3 | 保存成功引导 | 保存后弹窗提示"已保存到相册"，并提供"继续编辑"和"处理下一张"两个选项 |
| F-005-4 | 保存失败重试 | 权限不足或空间不足时给出明确提示和解决方案 |

**边界:**
- 微信小程序保存到相册的图片质量受系统限制，可能需要压缩处理
- 首次保存需用户授权相册写入权限
- HEIC 格式可能在部分 Android 手机上不兼容，需转码

---

### F-006 历史记录

**优先级**: P1（V1.1）

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-006-1 | 编辑历史列表 | 展示最近编辑记录（缩略图 + 使用的模板 + 时间），游客 ≤ 20 条，登录用户 ≤ 100 条 |
| F-006-2 | 重新编辑 | 点击历史记录可恢复当时的编辑状态继续修复 |
| F-006-3 | 清除历史 | 支持逐条删除或清空全部历史记录 |
| F-006-4 | 云端同步 | 登录用户的历史记录自动同步至云端，换设备后可恢复 |

**存储策略**: 与 F-004-A 登录系统联动，游客本地存储，登录用户云端存储 |

---

### F-007 分享与引导

**优先级**: P1（V1.1）

| 编号 | 需求描述 | 验收标准 |
|------|----------|----------|
| F-007-1 | 分享给好友 | 使用 `onShareAppMessage` 配置分享卡片，引导好友进入小程序 |
| F-007-2 | 分享到朋友圈 | 使用 `onShareTimeline` 支持朋友圈分享 |

## 三、非功能性需求

| 编号 | 类别 | 要求 |
|------|------|------|
| NF-001 | 性能 | 首页启动时间 ≤ 1.5s，编辑页图片加载 ≤ 2s |
| NF-002 | 包体积 | 主包 ≤ 1.5MB，为后续功能预留 0.5MB |
| NF-003 | 兼容性 | 微信基础库 ≥ 2.20.0，覆盖 iOS 12+ / Android 8+ |
| NF-004 | 隐私 | 图片处理全在客户端完成（不上传服务器），修复元数据不依赖网络 |

## 四、用户流程

### 4.1 核心路径（Happy Path）

```
[首页] → [从相册选图] → [进入修复页]
                           ↓
                    [查看当前参数]
                           ↓
            ┌──────────────┴──────────────┐
            ↓                              ↓
      [选择模板]                      [手动微调]
      → 一键套用                      → 逐字段修复
            ↓                              ↓
            └──────────────┬──────────────┘
                           ↓
                    [预览修复效果]
                           ↓
                    [保存到相册]
                           ↓
                    [完成/继续编辑]
```

### 4.2 模板创建路径

```
[修复页] → [当前配置满意] → [保存为我的模板] → [命名] → [确认]
                                                          ↓
                                                  [模板列表可见]
```

## 六、不做什么（明确边界）

1. **不做图片滤镜/调色** — 这不是修图工具，是参数修复工具
2. **不做云端存储原图** — 图片处理本地完成，不上传原图到服务器
3. **V1 不做社区功能** — 首版本不包含评论、点赞、模板广场，后续版本（P2）可考虑开放模板社区分享
4. **不做 AI 识别** — 不通过 AI 识别图片内容自动填充 EXIF
5. **不做视频参数** — 仅支持静态照片
